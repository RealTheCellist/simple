Product Promise

이 앱이 실제로 해결하는 것

사용자가 무엇을 보고 언제 반응할지를 제품 구조로 정의합니다. 빠르게 읽고, 빠르게 설정하고, 놓치지 않고 기록하는 경험이 핵심입니다.

01 Watchlist

행 단위 밀도

가격, 변동률, 추세선, 알림 상태를 한 줄에서 읽어 빠른 스캔을 가능하게 합니다.

02 Futures

시장 체온 파악

선물과 지수를 비교 가능한 구조로 보여 주어 시장 방향을 먼저 읽게 합니다.

03 Alerts

행동 위임

조건만 지정하면 앱이 백그라운드에서 감시하고 쿨다운까지 관리합니다.

04 History

판단 기록

Feature Spotlight

탭별 역할을 빠르게 확인

키보드 화살표로도 탭 전환이 가능합니다. 접근성을 포함해 실제 데모용 페이지로 사용할 수 있게 구성했습니다.

Watchlist는 즉시 스캔용 화면입니다.

종목 코드, 가격, 변화율, 미니 추세를 압축해서 보여 줍니다.

왼쪽 액센트 바로 상태 구분.
상단 마켓 스트립으로 시장 감각 동시 제공.

Futures는 시장 방향성 화면입니다.

빅카드와 2열 그리드로 지표 비교에 최적화합니다.

핵심 지표는 크게 강조.
나머지는 통일된 셀로 비교 속도 향상.
실패 시 안전한 fallback 표기.
Primary Value시장 체온 파악
UI MotifBig card + 2x3 grid
Data Hook60초 단위 갱신

Alerts는 입력 플로우가 핵심입니다.

종목 선택, 목표가, 조건을 짧은 흐름으로 등록합니다.

조건 색으로 above/below 구분.
활성 목록과 삭제 동작을 분리 표시.
탭 배지와 연동해 상태 인지 강화.

Build Notes

웹과 모바일의 구현 차이를 짧게 요약

상태 저장, 알림 API, 폴링 방식처럼 사용자 체감이 큰 차이에 집중합니다.

Storage

localStorage → AsyncStorage

동기 저장 대신 비동기 저장. 키 체계는 `sm_` 유지.

Notification

Browser API → Expo

권한 요청과 스케줄 방식이 달라 별도 유틸이 필요합니다.

Polling

Always-on → Focus-aware

활성 화면에서만 갱신해 배터리와 안정성을 확보합니다.
